# PRD — Prepare Ma Dot

## 1. Résumé du projet

**Prepare Ma Dot** est une plateforme web spécialisée dans la préparation, la fourniture et l'organisation de la dot africaine, avec un focus initial sur les cultures d'Afrique centrale.

La première version du produit doit permettre à l'équipe Prepare Ma Dot de démarrer rapidement ses activités en ligne avec une plateforme crédible, élégante, fonctionnelle et orientée conversion.

Cette V1 ne doit pas chercher à devenir une plateforme complexe dès le lancement. Elle doit être pensée comme un **site premium fonctionnel** combinant :

- une vitrine de marque crédible ;
- une boutique simple ;
- une présentation claire des services et packs ;
- un formulaire de demande personnalisée ;
- une prise de contact WhatsApp ;
- une base SEO saine ;
- une structure facilement améliorable après les premiers retours clients.

L'objectif est de lancer vite, vendre, apprendre, puis améliorer.

---

## 2. Contexte

Prepare Ma Dot souhaite aider les futurs mariés et leurs familles à préparer leur dot africaine plus facilement, y compris à distance.

Le service vise à résoudre plusieurs problèmes :

- difficulté à savoir quoi prévoir pour une dot selon les traditions ;
- manque de temps pour chercher les éléments nécessaires ;
- difficulté à coordonner les achats, la présentation, la livraison et l'organisation ;
- stress pour les familles vivant dans des villes ou pays différents ;
- besoin de préserver l'élégance culturelle et le respect des traditions.

La plateforme doit rassurer, expliquer, inspirer confiance et permettre une action immédiate : acheter, demander un accompagnement ou contacter l'équipe.

---

## 3. Objectifs business de la V1

La V1 doit permettre de :

1. Présenter Prepare Ma Dot comme une marque sérieuse, élégante et culturellement ancrée.
2. Expliquer clairement le concept à une personne qui découvre le service.
3. Présenter les catégories de produits liés à la dot.
4. Mettre en avant des packs ou services de préparation de dot.
5. Générer des demandes personnalisées via formulaire.
6. Rediriger facilement vers WhatsApp pour les échanges rapides.
7. Permettre la vente ou la précommande de produits/packs simples.
8. Préparer le terrain pour le référencement SEO.
9. Permettre à l'équipe de gérer les contenus de base après formation.
10. Lancer une première version exploitable sans attendre une plateforme complète.

---

## 4. Objectifs utilisateurs

Les utilisateurs doivent pouvoir :

- comprendre rapidement ce que propose Prepare Ma Dot ;
- découvrir les produits et éléments nécessaires à la dot ;
- consulter des packs ou offres d'accompagnement ;
- demander une préparation personnalisée ;
- contacter l'équipe par WhatsApp ;

- se rassurer sur le sérieux, l'élégance et la fiabilité du service ;
- acheter ou manifester une intention d'achat.

## 5. Cibles principales

### 5.1 Futurs mariés

Personnes préparant une dot et souhaitant être accompagnées pour gagner du temps, éviter les oublis et respecter les attentes familiales.

### 5.2 Familles

Parents, oncles, tantes, frères, sœurs ou proches impliqués dans l'organisation de la dot.

### 5.3 Diaspora africaine

Personnes vivant hors du pays concerné par la cérémonie et souhaitant préparer la dot à distance.

### 5.4 Partenaires potentiels

Fournisseurs, artisans, vendeurs de textiles, bijoutiers, créateurs, décorateurs, photographes, logisticiens ou acteurs culturels pouvant collaborer avec Prepare Ma Dot.

## 7. Périmètre de la V1

### Inclus dans la V1

La V1 doit inclure :

- un site responsive mobile-first ;

- une page “Demande personnalisée” ;
- une page “À propos” ;
- une page “Contact” ;
- un formulaire de demande avec possibilité d'ajouter des informations détaillées ;
- idéalement un champ d'upload de document ou image ;
- un bouton WhatsApp visible ;
- une intégration réseaux sociaux ;
- un système de gestion de contenus simple ;
- les bases SEO ;
- les statistiques de visite ;

### Exclu de la V1

Ne pas intégrer dans la V1 :

- marketplace multi-vendeurs ;
- comptes clients avancés ;
- espace partenaire ;
- application mobile ;
- recommandation automatique complexe ;
- devis automatique intelligent ;
- gestion avancée de logistique ;
- tableau de bord analytique complexe ;
- paiement international avancé ;
- système de gestion des cérémonies complet ;
- workflow interne de production trop détaillé.

Ces éléments pourront être étudiés en V2 après validation du marché.

## 9. Fonctionnalités détaillées

## 9.1 Responsive mobile-first

Le site doit être conçu prioritairement pour mobile, car les prospects utiliseront probablement WhatsApp, Instagram et mobile comme principaux points d'entrée.

Exigences :

- navigation simple ;
- CTA visibles ;
- chargement rapide ;
- textes lisibles ;
- images optimisées ;
- formulaire facile à remplir sur téléphone.

---

## 9.2 Gestion des produits

Le système doit permettre de gérer les produits :

- créer un produit ;
- modifier un produit ;
- supprimer ou masquer un produit ;
- ajouter photos ;
- définir catégorie ;
- définir prix ;
- définir statut : disponible, indisponible, sur demande.

Pour la V1, il n'est pas obligatoire d'avoir une gestion de stock avancée.

---

## 9.3 Gestion des packs/services

Le système doit permettre d'afficher les packs/services.

Chaque pack doit contenir :

- nom ;
- description courte ;
- détails inclus ;
- prix ou mention “sur devis” ;
- image optionnelle ;
- CTA.

---

## 9.4 Formulaire de demande personnalisée

Le formulaire doit :

- enregistrer les demandes ;
- envoyer une notification email ;
- être protégé contre le spam ;
- être simple sur mobile ;
- afficher un message de succès clair.

Option technique : sauvegarder les demandes dans une base de données ou les envoyer via un service de formulaire.

---

## 9.5 WhatsApp

Le site doit intégrer un bouton WhatsApp :

- visible dans le header ou en bouton flottant ;
- présent sur les fiches produits ;
- présent sur les pages packs ;
- message prérempli selon le contexte si possible.

Exemples de messages préremplis :

- “Bonjour, je souhaite préparer une dot et j'aimerais être accompagnée.”
- “Bonjour, je suis intéressé(e) par ce produit : [nom du produit].”
- “Bonjour, je souhaite recevoir une proposition pour le Pack Premium.”

---

## 9.6 Paiement en ligne

La demande initiale mentionne le paiement en ligne. Pour la V1, deux niveaux sont possibles :

### Option simple

- pas de paiement direct au lancement ;
- les commandes ou demandes se finalisent via WhatsApp ;
- paiement manuel via Orange Money ou autre moyen.

### Option avancée V1

- intégration d'un paiement en ligne simple ;
- paiement pour certains produits ou packs ;
- redirection vers un prestataire de paiement adapté.

Recommandation : démarrer avec une option simple ou semi-manuelle si le budget et les délais sont limités.

---

## 9.7 Multilingue

La demande mentionne français + anglais. Pour la V1, prévoir l'architecture permettant d'ajouter l'anglais, mais ne pas bloquer le lancement si tout le contenu anglais n'est pas prêt.

Priorité : version française complète.

Option : anglais en phase 2.

## 10. Design et identité visuelle

## 10.1 Direction artistique

Le design doit être :

- chaud ;
- humain ;
- africain ;
- ancré culturellement ;
- premium ;
- minimaliste ;
- élégant ;
- non surchargé ;
- non générique ;
- non “trop IA”.

## 10.2 Couleurs

Couleurs souhaitées :

- bordeaux ;
- or ;
- beige.

Palette suggérée :

- Bordeaux profond : `#6E1F2F`
- Or doux : `#C8A24A`
- Beige clair : `#F5EBDD`
- Ivoire : `#FFF9F0`
- Brun texte : `#2F1E1A`
- Bordeaux foncé : `#3A111A`

Ne pas utiliser des couleurs trop criardes.

## 10.3 Typographie

Recommandation :

- une police élégante pour les titres ;
- une police très lisible pour les paragraphes.

Exemples :

- Titres : Playfair Display, Cormorant Garamond ou Libre Baskerville ;
- Texte : Inter, Lato, Manrope ou Source Sans 3.

## 10.4 Images

Utiliser des images :

- réalistes ;
- chaleureuses ;
- culturelles ;
- premium ;
- avec textiles, paniers, accessoires, bijoux, familles, cérémonies ou détails de préparation.

Éviter :

- images trop corporate ;
- rendus IA trop visibles ;
- visuels surchargés ;
- clichés culturels exagérés ;
- photos de mauvaise qualité.

## 12. Parcours utilisateur principal

### Parcours 1 — Demande personnalisée

1. L'utilisateur arrive sur la page d'accueil.
2. Il comprend le service.
3. Il clique sur “Préparer ma dot”.
4. Il remplit le formulaire.
5. Il reçoit une confirmation.
6. L'équipe le contacte par WhatsApp.

### Parcours 2 — Découverte produit

1. L'utilisateur arrive via Google ou Instagram.
2. Il ouvre la boutique.
3. Il consulte les catégories.
4. Il ouvre une fiche produit.
5. Il clique sur WhatsApp ou demande un devis.

### Parcours 3 — Diaspora

1. L'utilisateur cherche à préparer une dot à distance.
2. Il lit la page “Comment ça marche”.
3. Il consulte le Pack Premium / À distance.
4. Il remplit le formulaire avec son pays, son budget et sa date.
5. L'équipe le contacte pour cadrer le besoin.

## 15. Critères d'acceptation

La V1 est considérée prête si :

- le site est en ligne sur le domaine Prepare Ma Dot ;
- toutes les pages principales sont disponibles ;
- le site est lisible et agréable sur mobile ;
- les CTA fonctionnent ;
- WhatsApp fonctionne ;
- le formulaire de demande fonctionne ;

## 18. Priorité absolue

La priorité n'est pas de créer la plateforme parfaite.

La priorité est de créer une première version crédible, belle, simple et fonctionnelle qui permet à Prepare Ma Dot de :

- expliquer son offre ;
- recevoir des demandes ;
- vendre ses premiers produits ou packs ;
- apprendre des vrais clients ;
- évoluer sur des bases solides.
